如何用linux操作系统创建一个c程序

1.首先我们可以先创建一个专属文件夹,然后在该文件夹中创建一个专属c语言文件(由于linux截屏比较麻烦,用手写代码)

$ mkdir first

cd first

touch mkdir first.c

vi first.c

3.进入first.文件,开始编辑c语言

按i进入编辑

int main(){

printf(“hello,world”);

}

4.简单代码编写完成

按esc退出编写模式

按shift+:进入底部命令

输入wq保存内容并退出该.c文件

回到操作系统

5.使用gcc来编译刚才的代码

gcc first.c -o first.exe

此行代码是将编译好的结果用一个新的可执行文件来接受

下次运行代码直接执行该.exe文件即可

即./first.exe

此命令意为执行此目录下的fiest.exe文件

输出结果hello,world

最后可以输入exit以退出linux

对进程的了解

关于fork函数

1.在父进程中,fork函数创建子进程的PID(进程的ID);

2.在子进程中,fork返回0;

3.未能创建,fork返回负值;

4.我们可以通过fork的返回值判断当前进程;

为什么fork要给子进程返回0,给父进程返回子进程的pid?

返回不同的返回值,是为了区分让不同的执行流执行不同的代码块.父进程接收子进程pid是因为父进程可能有多个子进程,为了区分所以用他们特有的pid来作为返回值,而子进程直接用getppid()这个函数就能获得父进程的pid,所以子进程的进程返回就是0;

进程状态

常见的进程状态有运行,挂起,阻塞,就绪,等待,死亡等;

运行状态:

进程已经随时准备好被调度;

运行队列:

每个cpu都会维护一个自己的运行队列(一个cpu,一个运行队列);

运行状态下存在两种情况:正在运行的和在运行队列中的;

睡眠状态:

S状态:可中断睡眠(浅度睡眠)状态;

操作系统在资源紧缺时,可以直接杀死该状态的进程;

[!NOTE]

1.printf是从外设上取数据,循环打印时,不一定每次打印时,外设都准备好了,因为cpu比外设速度快的多,所以cpu会等待外设;

2.因为在运行该程序时,发现进程状态时S,表明大部分时间进程还是在等待的,而不是在运行(可以在某个瞬间看到它是R状态,但概率很低);

3.大部分进程都会与外设有关,会有等待的状态;

深度睡眠

D状态:不可中断睡眠;操作系统不会杀D状态的进程;

深度睡眠产生的原因:

会导致将数据丢掉;

所以增加一个深度睡眠状态的进程,用来阻止进程被杀死,即使实在资源紧缺的情况下;

暂停状态

T状态:暂停进程;

可以手动暂停和继续进程;

查看kill信号:

其中,18即进程继续,19为进程暂停;

被追踪状态

t状态:被追踪的暂停状态;当进程处于被追踪状态时,进程暂停,等一等追踪我的那个东西;

僵尸状态

Z状态:进程已经执行完毕,但并未获得进程退出的相关信息时的状态;

管道通信

代码

1.要求:编写程序实现父子进程的管道通信。父进程建立一个管道，其二个子进程P1和P2分别向管道中各写入一句话，父进程分别从管道中读出二个来自子进程的信息并显示之;

2.代码:

代码解析:

头文件包含

  • unistd.h：提供对POSIX操作系统API的访问，包括管道和进程控制函数。

  • signal.h：提供信号处理相关的函数和宏。

  • stdio.h：提供标准输入输出函数，如printf。

  • stdlib.h：提供标准库函数，如exit。

    主函数

  • int fd[2];：定义一个文件描述符数组，用于管道的两端。

  • pipe(fd);：创建一个管道，fd[0]用于读端，fd[1]用于写端。

    创建第一个子进程

  • while((p1 = fork()) == -1);：不断尝试创建子进程，直到成功。如果fork返回-1，表示创建失败。

  • if (p1 == 0)：判断是否为子进程。fork返回0表示当前是子进程。

    • close(fd[0]);：关闭管道的读端。
    • lockf(fd[1], 1, 0);：对管道的写端加锁，防止其他进程同时写入。
    • char *message1 = "child process P1!";：定义要写入管道的消息。
    • write(fd[1], message1, 50);：将消息写入管道的写端。
    • lockf(fd[1], 0, 0);：解锁管道的写端。
    • close(fd[1]);：关闭管道的写端。

    创建第二个子进程

  • else：表示当前是第一个子进程的父进程。

  • while((p2 = fork()) == -1);：不断尝试创建第二个子进程，直到成功。

  • if (p2 == 0)：判断是否为第二个子进程。

    • close(fd[0]);：关闭管道的读端。
    • lockf(fd[1], 1, 0);：对管道的写端加锁。
    • char *message2 = "child process P2!";：定义要写入管道的消息。
    • write(fd[1], message2, 50);：将消息写入管道的写端。
    • lockf(fd[1], 0, 0);：解锁管道的写端。
    • close(fd[1]);：关闭管道的写端。

    父进程读取消息

  • else：表示当前是两个子进程的父进程。
    • close(fd[1]);：关闭管道的写端。
    • char message[100];：定义一个缓冲区用于存储从管道读取的消息。
    • wait(NULL);：等待任意一个子进程结束。
    • read(fd[0], message, 50);：从管道的读端读取消息。
    • printf("%s\n", message);：打印读取到的消息。
    • wait(NULL);：再次等待另一个子进程结束。
    • read(fd[0], message, 50);：再次从管道的读端读取消息。
    • printf("%s\n", message);：打印读取到的消息。
    • close(fd[0]);：关闭管道的读端。
